# Security Policy

## Reporting a vulnerability

Please report security issues for this website or repository by opening a private GitHub security advisory, or by contacting the repository owner directly if that is not available.

Do not disclose exploitable issues publicly until they have been assessed.

## Scope

Security reports may include issues in the static site, build/deploy workflow, dependencies, or repository configuration.

Food-safety corrections are welcome as normal issues or pull requests, but they are not handled as software security vulnerabilities unless they also affect the website or repository security.
